# Task 06 - Create Package & Pricing CRUD

## Goal
Allow vendors to define service packages and pricing for customers to review.

## Scope
- Create package entities linked to vendor profiles
- Support package title, description, price, and inclusions
- Build create, update, delete, and list functionality
- Prepare pricing data for future booking flows and comparisons

## Deliverables
- Package pricing schema
- CRUD APIs or server actions
- Vendor-facing package management UI
- Validation for prices and required fields

## Acceptance Criteria
- Vendors can create multiple service packages
- Package details can be edited and deleted safely
- Pricing data is displayed clearly and stored accurately
- Package records can be shown on public profiles and comparison views
