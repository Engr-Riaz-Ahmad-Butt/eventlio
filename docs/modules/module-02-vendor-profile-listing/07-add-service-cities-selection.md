# Task 07 - Add Service Cities Selection

## Goal
Allow vendors to define the cities or service areas where they operate.

## Scope
- Create service area or city selection structure
- Support one or many cities per vendor
- Provide a searchable or selectable list in the UI
- Prepare data for marketplace city filtering

## Deliverables
- Database relation for vendor service areas
- Frontend city selection UI
- Persistence layer for save and update actions
- Data shape that supports search and filters later

## Acceptance Criteria
- Vendors can select one or more service cities
- Saved cities load correctly on profile edit
- City data can be reused for discovery filters
- Invalid or duplicate assignments are prevented
