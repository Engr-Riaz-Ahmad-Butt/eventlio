# Task 08 - Create Onboarding Wizard Screens

## Goal
Guide newly registered users through the initial setup flow based on their selected role.

## Scope
- Build step-by-step onboarding screens
- Create a vendor onboarding path
- Create a client onboarding path
- Save progress between steps
- Show completion state and next actions

## Deliverables
- Shared onboarding layout
- Vendor onboarding screens
- Client onboarding screens
- Progress persistence logic

## Acceptance Criteria
- New users are redirected into onboarding after signup when needed
- Users can move between steps reliably
- Progress is saved if the user leaves and returns
- Completion updates the user onboarding status
